源码编译Openssl时遇到的疑难杂症

环境

  • centos 6.5
  • openssl 1.1.1d

安装步骤

#!/bin/bash
curl -O https://www.openssl.org/source/openssl-1.1.1d.tar.gz
tar -C /usr/local/src/ -xf openssl-1.1.1d.tar.gz
cd /usr/local/src/openssl-1.1.1d
./config --prefix=/usr/local/openssl --openssldir=/usr/local/openssl shared zlib
make 
make test
make install
mv /usr/bin/openssl /usr/bin/openssl.bak
mv /usr/include/openssl/ /usr/include/openssl.bak
ln -s /usr/local/openssl/bin/openssl /usr/bin/openssl
ln -s /usr/local/openssl/include/openssl/ /usr/include/openssl
echo "/usr/local/openssl/lib" >> /etc/ld.so.conf

openssl: error while loading shared libraries: libssl.so.1.1: cannot open shared object file: No such file or directory

这是由于openssl库的位置不正确导致的, 创建软链接解决. 

ln -s /usr/local/lib64/libssl.so.1.1 /usr/lib64/libssl.so.1.1
ln -s /usr/local/lib64/libcrypto.so.1.1 /usr/lib64/libcrypto.so.1.1

Can’t locate Test/Harness.pm in @INC

yum install perl-CPAN
perl -MCPAN -e shell

一路yes, 到cpan[1]

install Text::Template

直到安装完成, 退出, 再make test

note: 晚上加班安装的openssl, 比较着急, 没有仔细看openssl官网

整理360+条书签到75条

最近觉得自己书签数量太多, 内容太杂, 清理了一波. 觉得自己有些地方做的不好

  1. 我有一个稍后看的文件夹, 下面的网页太多了, 都是些放了, 但没时间看的, 或者其实没什么用的网页
  2. 收藏了一些网课/教程, 其实没必要, 等需要了再搜索, 一样的效果
  3. 收藏了比较多的科普/教程文章,
  4. 收藏了一些在某一时刻解决了问题的文章, 其中有一些掌握了, 仍然收藏了. 有一些没整理到笔记中.
  5. 习惯性按ctrl + D
  6. 收藏了各种各样的库/框架/官网/手册, 这类文档往往有时效性, 不如直接搜索.
  7. 收藏了上一点中的中文翻译.
  8. 各种各样的工具使用技巧/速查表
  9. 有的内容, 不会影响我的任何决定, 可能纯粹是谈资, 有的就只是单纯的放在那里, 没动过

归结为:

  • 碎片化
  • 缺少整理
  • 习惯按ctrl D 的坏习惯
  • 网上的内容大多来自同一个地方, 只是被复制/翻译/修改, 分成了很多份. 我还没辨认出来, 没学到核心, 导致看什么都很新鲜

还有一个点就是, 不知道从什么时候开始, 我就再也没使用过onetab这个插件了, 可能那段时间用ctrl + D, 代替了这个插件.
导出的书签有网页被收藏的时间戳, 可以调查一下.

因此, 清理了很多这类网页, 减少了我看到书签量大的恐惧, 增加重新访问书签的频率.
同时, 也增加了很多需要整理的内容

现在书签内还有很多博主的博客链接, 应该还需要转换成订阅源.

疫情新年远程办公感受

这篇由疫情期间在家远程办公, 每天onenote的记录得来. 主要记一下远程办公的感受:

  1. 个人的变化
  2. 与其他程序员远程办公的对比
  3. 与其他职业的人的对比

个人的变化

家在南方沿海, 2月4日 左右无阳光, 冷得发抖. 加上远程办公, 网络连接真不能算流畅, 远程连接有几个IDE的快捷键不能用, 人处于一个比较烦躁的状态

第一次在家工作, 特别地, 家里也没有专门工作的房间, 在卧室里, 就总想睡, 想趴下. 没有办公的状态, 归结为心还没沉下来.

作息不同, 又可能要24 hours on call. 部门内作息时间交叉, 也不像在公司, 下班就下班了, 一开始还会码到睡觉前.

后来从我的onenote记录来看, 情况好得多, 我在每一天, 每一页的记录都增加了一串流水账, 这也是从别人那里学来的. 

这个笔记的顶部右边放作息
分别插入: 保证8小时即可.
早上上班, 早上下班
中午上班, 下午下班
晚上上班, 晚上下班

中间在细致到扣除吃饭, 睡觉, 忽然出门等等.

再加上, 这段时间内只专注于代码, 没有平常在公司碰到的杂活. 没有公司那么吵, 更加的安静和专注.

与其他程序员远程办公的对比

有的公司在疫情期间, 连远程办公都没法提供, 有的不需要上班, 有的心里一直想着去上班.

跟他们对比, 我有点幸运, 有点羡慕, 有点惭愧和不解.

我在的部门比较小, 人也不多, 办公的那段时间, 工作内容也比较直白. 论坛上其他人提到的交流不方便, 还没能深刻地体会到.
以及家里比较吵闹, 有小孩要照顾等等, 这些也没遇到, 相反, 疫情让父母呆在家里安心看电视, 追剧. 到村口帮忙疫情值班. 整体来说, 年后日子比较平静, 也比较少串门.
在家里也不用担心吃饭, 卫生的打扫. 父母也有个人使唤.

与其他职业的对比

有台电脑, 有远程连接就能办公. 这在我们村子那是件新奇的事情. 有的不发工资, 有的不知道什么时候上班, 有的降薪, 有的直接就开除了.

还有一个点是有些人由于事情太突然, 一下失去了现金流, 同时又有各种各样的贷款, 债务. 这个也是我还不能体会到的.

记Hexo + Github Pages 建博客

今天心血来潮, 想要重新用github建立个博客. 官方的教程不错. 这里记录下遇到的问题

通过两仓库或两分支的操作, 把博客内容区和工作区分开, 同时跟踪和保留工作现场(备份).

这里我选择两仓库, 工作区私有, 内容区公开. 使用官方的工具, 非常方便.

搜索引擎收录问题, 谷歌搜索 

site: https://lonenso.github.io/


这时候网站没有被收录, 搜索结果中有个 google search console, 根据提示完成网站认证的添加.

我选择的是将html文件放到站点下, 使得https://lonenso.github.io/xxxx.html 能够访问到该文件.

也就是说, 本地https://localhost:4000/xxxx.html 能够访问到该原文件, 不是被渲染后的, 配置skip_render.

接下来增加评论功能, 注册disqus, 根据指引, 最终需要手动将html代码嵌入页面中.

hexo的默认主题landscape下的article.ejs, 文件末尾有一部分和disqus给的html代码很像, 把disqus给的html代码 调整放到这里, 并且把写死的disqus_shortname替换成变量

<% if (!index && post.comments && config.disqus_shortname){ %>
<section id="comments">
...
</section>

在_config.yml中配置disqus_shortname即可
