记Hexo + Github Pages 建博客

今天心血来潮, 想要重新用github建立个博客. 官方的教程不错. 这里记录下遇到的问题

博客内容和工作区结构组织

通过两仓库或两分支的操作, 把博客内容区和工作区分开, 同时跟踪和保留工作现场(备份).

这里我选择两仓库, 工作区私有, 内容区公开. 使用官方的工具, 非常方便.

搜索引擎收录

谷歌搜索 

site: https://lonenso.github.io/


这时候网站没有被收录, 搜索结果中有个 google search console, 根据提示完成网站认证的添加.

我选择的是将html文件放到站点下, 使得https://lonenso.github.io/xxxx.html 能够访问到该文件.

也就是说, 本地https://localhost:4000/xxxx.html 能够访问到该原文件, 不是被渲染后的, 配置skip_render.

disqus 评论集成

接下来增加评论功能, 注册disqus, 根据指引, 最终需要手动将html代码嵌入页面中.

hexo的默认主题landscape下的article.ejs, 文件末尾有一部分和disqus给的html代码很像, 把disqus给的html代码 调整放到这里, 并且把写死的disqus_shortname替换成变量

<% if (!index && post.comments && config.disqus_shortname){ %>
<section id="comments">
...
</section>

在_config.yml中配置disqus_shortname即可

添加rss feed

hexo-generator-feed
我在hexo下添加博客功能常犯的一个错误就是把主题的配置和博客的配置搞混

npm install hexo-generator-feed --save

添加以下配置到博客的配置_config.yml下 

feed:
  type: atom
  path: atom.xml
  limit: 20
  hub:
  content:
  content_limit: 140
  content_limit_delim: ' '
  order_by: -date
  icon: icon.png
  autodiscovery: true
  template:

添加公式支持

npm install hexo-math --save

开箱可用, 不需要再加配置了
hexo-math 有点问题, 具体是自己遇到的#120
后来我理了下逻辑

mkdir newfolder 
cd newfolder && hexo init
npm install hexo-math --save
hexo new post a
echo $a=b+c$ >> source\_posts\a.md
hexo clean && hexo s

这时打开浏览器, 看不到效果.
于是我根据#26

npm install hexo-renderer-mathjax --save
npm list hexo-renderer-mathjax

会发现此时还没有hexo-renderer-mathjax这个库. 大概原因是postinstall触发安装了hexo-inject.
再执行一次

npm install hexo-renderer-mathjax --save
hexo clean && hexo s

这时候, 打开浏览器就有了. 至于为什么本地看是https, 发布后是http就不清楚了.

既然在#26中有那么多人遇到了使用上的问题, 不如不用这个插件了.

改为手动修改html代码, 选择页面中加载mathJax, 在after-footer.ejs中添加代码

<script type="text/x-mathjax-config">
  MathJax.Hub.Config({
      tex2jax: {
          inlineMath: [ ["$","$"], ["\\(","\\)"] ],
          skipTags: ['script', 'noscript', 'style', 'textarea', 'pre', 'code'],
          processEscapes: true
      }
  });
  MathJax.Hub.Queue(function() {
      var all = MathJax.Hub.getAllJax();
      for (var i = 0; i < all.length; ++i)
          all[i].SourceElement().parentNode.className += ' has-jax';
  });
</script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/mathjax/2.7.1/MathJax.js?config=TeX-AMS-MML_HTMLorMML"></script>

同时因为自己乱找解决办法, 决定清理依赖. 其实大体都是注入mathJax的支持, 只不过想用插件把配置统一管理到站点下的配置文件里. 同时有的人可能想要切换主题, 因此一般也不直接修改theme下的代码.
这里就不纠结了, 直接修改代码, 节省时间.

npm uninstall hexo-inject
npm uninstall hexo-math
npm uninstall hexo-renderer-mathjax
rm -rf node_modules
npm install

添加canvas-nest

我还挺喜欢这个动效的, 但是好像有个css层级问题, 导致看不见效果. 后来发现应该是个颜色问题. 页面中#wrap这个标签还包含了.outer.
wrap的颜色和outer的颜色会叠加. 而原来之所以没有明显的颜色界线是因为默认颜色#eee, alpha通道为1. 不透明, 导致一开始添加canvas-nest时没有直接看到效果.
而调整alpha通道, 由于两个标签(图层)叠加alpha通道, 产生了颜色界线.


通过区别#wrap和.body的颜色, 调整#wrap的alpha通道为0解决
