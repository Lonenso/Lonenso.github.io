Note:
建堆: O(n) 次调用堆化, 粗略O(nlgn), 但能得到更紧确的时间复杂度O(n)
堆排序: 先建堆, 然后弹出堆顶元素, 替换堆末尾的元素到顶部, 堆化.
插入元素: 插入无限小元素到末尾, 然后调用HEAP-INCREASE-KEY. 完成插入
提取最大值: 弹出堆顶, 替换成末尾元素, 然后下沉
增加关键字的值: 值增加后, 节点上浮.

6.5-3

minHeap

6.5-4

MAX-HEAP-INCREASE(A, i, key) 中要求key的值比A[i]的大

6.5-6

先移动和变换, 找到插入的位置再赋值key

HEAP-INCREASE-KEY(A, i, key)
if key < A[i]
    error "new key is smaller than current key"
while i > 1 and A[PARENT(i)] < key
    A[i] = A[parent(i)]
A[i] = key

6.5-7

用插入的顺序的值构建最小堆, 另存储元素实际的值
优先队列(最小堆)实现先进先出队列: 先进先出, 插入的顺序是增加的
优先队列(最小堆)实现栈: 相反, 插入的顺序是减小的.

6.5-8

(最大堆)
都是把要删除的, 要提取的元素和末尾元素替换, 然后重新堆化(Heapify).
但是为了保证删除后, 还能维持堆的性质. (与末尾元素相比). 当删除的元素较大, 得上浮. 当删除的元素较小, 得下沉

操作MAX-HEAPIFY 是 O(lgn), HEAP-INCREASE-KEY 也是O(lgn). 

MAX-HEAP-DELETE(A, i)
if A[i] < A[A.heap-size]
    A[i] = A[A.heap-size]
    MAX-HEAPIFY(A, i)
else
    HEAP-INCREASE-KEY(A, i, A[A.heap-size])
A.heap-size = A.heap-size - 1
